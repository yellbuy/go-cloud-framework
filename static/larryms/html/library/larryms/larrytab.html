框架核心组件之菜单选项卡组件larryTab

larryTab组件简介：larryTab.js主要实现后台界面框架中菜单初始化与tab选项卡相关功能，依赖于larryms组件，实现以tab选项卡方式加载iframe，选项卡100种随机动画加载，框架刷新状态保存，切换刷新、菜单模式灵活切换（顶部菜单导航开启或关闭），自动定位伸展，页面选项卡方式打开、iframe内页任意选项卡方式打开、与导航菜单自动跟随一一对应等等，围绕后台界面展现多种功能灵活组合搭配，快速搭建后台界面。

larryTab组件方法介绍

名称 参数 用法 说明
set options
描述： options 参考开发者文档介绍 		larryTab = layui.larryTab({tab_elem: '#larry_tab',tabMax: 20,spreadOne: true}); larryTab组件初始化 完成后台主体界面的初始化，设定相关必要参数，当然也可通过下面介绍的menuSet与tabSet方法分别细化配置
menuSet options 描述： options 参考开发者文档介绍
支持针对'data', 'url', 'type', 'cached', 'spreadOne', 'top_menu', 'topFilter', 'left_menu', 'leftFilter'等参数的个性化配置设定 		larryTab.menuSet(options) larryTab.menuSet方法也是LarryMS框架在iframe版本中后台界面的核心方法,用于导航菜单初始化，以json数据格式前后端交互 ，如果是网站前台或后台SPA版本中该组件用不到
tabSet options 描述： options 参考开发者文档介绍
支持针对'tab_elem', 'tabFilter', 'tabSession', 'closed', 'tabMax', 'autoRefresh', 'tabShow', 'pageEffect','isPageEffect'等参数的个性化配置设定 		larryTab.tabSet(options) tabSet用于tab选项卡相关功能的设定，如动画加载的开启关闭，选项卡默认打开个数，是否切换刷新，是否保持状态等等，具体以开发者文档详细示例为准。
tabAdd data
描述：data具体参数参看larryMS框架文档		 默认已于框架菜单功能中，用于tab选项卡方式的打开新增，larryTab.tabAdd(data); 支持子页面或框架非导航位置的选项卡新增或自动定位，可使用属性：larry-tab="[iframe|page]" 在选项卡中打开页面
tab系列方法 tabChange、tabDelete、tabAuto、autoRefresh、getTabId、contentAddTab、tabInit、render等等二十多个方法已在框架相关功能上体现，用户无特殊开发需求情况下可以忽略，更为灵活的使用与结构变化时可以使用，提供于LarryMS框架文档中
