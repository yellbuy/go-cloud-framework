关于LarryMS框架用户自定义组件扩展使用说明

为了满足LarryMS框架使用者在项目开发中足够的灵活扩展，从beta2版本开始新增了用户自定义扩展功能（虽然larryms框架已自带大量优秀JS库，layui自身组件也在不断完善，增加底层支持用户自定义组件库会让larryms框架更为易用和实用），下面对用户自定义扩展组件在larryMS框架中的使用做下简要说明：

自定义组件扩展功能使用说明

1、用户自定义扩展组件功能的定义

为了让LarryMS框架使用更为灵活更为使用，从beta2版本开始新增用于用户自定义组件扩展的底层支持，用户只需按照layui文档中定义的组件扩展规范扩展的组件均可加入LarryMS框架用户自定义的扩展目录中，即可像layui自带组件一样使用，无需别名，繁琐的路径配置，亦无任何延迟等开销，真正的开箱即用。该扩展功能的支持可以让用户对框架的使用更为灵活实用。

2、组件的引入与使用

虽然在LarryMS框架中已提供丰富的扩展JS库，但也不能面面俱到，对用户自定义组件扩展灵活支持可以让项目开发更便捷高效，具体的使用可参考右侧关于用户自定义组件扩展的代码示例。

3、自定义组件的规范与layui扩展平台组件引用说明

layui模块定义方法：layui.define([mods], callback)
通过该方法可定义一个 Layui模块。参数mods是可选的，用于声明该模块所依赖的模块。callback即为模块加载完毕的回调函数，它返回一个exports参数，用于输出该模块的接口。
跟Requirejs最大不同的地方在于接口输出，exports是一个函数，它接受两个参数，第一个参数为模块名，第二个参数为模块接口，当你声明了上述的一个模块后，你就可以在外部使用了，demo就会注册到layui对象下，即可通过 layui.demo() 去执行该模块的接口。 

layui.define(function(exports){
  //do something
  
  exports('demo', function(){
    alert('Hello World!');
  });
});
亦可引用layui扩展平台或其他已封装好的模块组件到扩展目录下；如右侧示例中添加对应组件模块及路径到配置文件conf.json中。
自定义组件中若有css文件需引入的，可直接在组件中使用layui.link()方法添加

更多详情内容请查看LarryMS框架开发者文档

自定义组件扩展使用代码示例

1、全局配置文件开启自定义扩展

#...省略
thirdExtend:true,//开启用户自定义扩展组件目录 false时关闭
thirdDir: 'thirdlib/',//用户自定义扩展组件目录，thirdlib为用户自定义目录名称（可以任意修改名称但路径需要在larryms目录下）

2、例如用户自定义扩展组件的目录为 thirdlib,则可以将用户自己写的扩展组件或layui扩展平台下的组件，或其他符合layui组件扩展规范的组件放入该目录下，并添加配置文件conf.js；假设用户在自定义组件目录中添加了custom1.js 和cus目录及cus/custom2.js组件

{
    "custom1": "custom1",
    "custom2":"cus/custom2"
}

如上扩展组件配置文件，分别定义了custom1和custom2组件，键值对中键为组件名称，值为组件所在路径，均相对于用户扩展组件目录而设定

3、使用用户自定义的组件(示例)

layui.use(['custom1','custom2'],function(){
    var custom1 = layui.custom1,
        custom2 = layui.custom2;
    # your code ...
});

使用起来是不是很方便，在任何位置都可以像layui自带组件一样使用，无需别名方式，无需关注底层实现，加载无任何延迟，快速而高效，这就是larryms框架的方便之处
